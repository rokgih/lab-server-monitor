# Lab Server Monitor

Live-ish dashboard of CPU/memory/GPU usage across multiple master servers and
their GPU nodes. Each master pushes a JSON snapshot to this repo every 5 min;
GitHub Pages re-deploys on push; the page auto-refreshes every 30 seconds.

```
master-1: cron */5  →  collect.py  →  data/master-1.json  →  git push
master-2: cron */5  →  collect.py  →  data/master-2.json  →  git push
                                          ↓
                              this repo (lab-server-monitor)
                                          ↓ Pages auto-deploys
                              https://<account>.github.io/lab-server-monitor/
```

## What's collected per node
- CPU %, core count
- Memory used / total / %
- 1/5/15-min load average
- GPU (via `gpustat --json`) — utilization, memory, temperature, processes
- Process owners are anonymized as `u_xxxxxx` (sha256 prefix)

## One-time GitHub setup

1. Create this public repo (e.g. `<account>/lab-server-monitor`) and push the
   contents of this directory.
2. **Settings → Pages → Source: GitHub Actions**.
3. Issue a fine-grained PAT for pushing snapshots (one shared PAT for all masters
   is fine):
   - Repository access: this repo only
   - Permissions → **Contents: Read and write**
   - Expiration: 1 year
4. Save the PAT — you'll need it on every master.
5. Trigger the deploy workflow once: Actions tab → `deploy` → Run workflow.

> **Repo public vs private**: public is recommended (free Actions minutes,
> simpler). The data shown is already going public via the Pages URL anyway.

## Per-master setup (do this on each master server)

### 1. Install dependencies

```bash
# On the master:
pip install --user psutil pyyaml gpustat   # gpustat optional if no GPUs

# On every node the master will SSH to:
pip install --user psutil gpustat
```

### 2. SSH key auth from master to nodes

`collect.py` SSHs to each node non-interactively. Confirm
`ssh node-01 echo ok` works without a password prompt before scheduling.

### 3. Clone the repo and make config

```bash
# Pick a stable location, e.g. ~/lab-server-monitor
git clone https://<account>:<PAT>@github.com/<account>/lab-server-monitor.git
cd lab-server-monitor
cp nodes.yaml.example nodes.yaml
$EDITOR nodes.yaml          # set master_name and node list
```

The `nodes.yaml` is gitignored — never commit it.

### 4. Test it once

```bash
python3 collect.py --config nodes.yaml --out data/$(grep '^master_name' nodes.yaml | awk '{print $2}').json
cat data/*.json | head -40
```

You should see CPU/mem numbers and (for GPU nodes) GPU utilization.

### 5. Set up cron (every 5 min)

The repo includes `run.sh` — a wrapper that runs `collect.py`, commits, and
pushes. Make it executable, test it once, then schedule it:

```bash
chmod +x run.sh
./run.sh                # smoke test — should produce no errors
crontab -e
```

Add **a single line** (cron does NOT support backslash continuation):

```cron
*/5 * * * * /home/USER/lab-server-monitor/run.sh >> /tmp/lsm.log 2>&1
```

Replace `/home/USER/lab-server-monitor` with the absolute path to your clone.
Logs append to `/tmp/lsm.log`. If the snapshot didn't change since last run
(e.g. nothing happened), `run.sh` skips the commit silently.

### 6. Verify

After 5 min, check:
- `data/master-1.json` exists in the repo on GitHub
- Actions tab shows a green deploy
- `https://<account>.github.io/lab-server-monitor/` shows your master's section

## Local development

```bash
# Generate fake data and serve the page:
python3 mock_data.py            # writes data/master-mock.json (see below)
python3 -m http.server 8765     # then add manifest.json manually:
echo '{"files":["master-mock.json"]}' > manifest.json
# Open http://localhost:8765
```

(In production, `manifest.json` is generated by the deploy workflow.)

## Files

- `collect.py` — master-side collector (cron target)
- `nodes.yaml.example` — config template; copy to `nodes.yaml` per master
- `index.html` / `dashboard.js` / `style.css` — frontend
- `.github/workflows/deploy.yml` — Pages deploy on every push
- `data/<master>.json` — snapshots, written by `collect.py`, committed by cron

## Troubleshooting

- **All nodes "unreachable"** — SSH key auth not set up. Test
  `ssh -o BatchMode=yes node-01 echo ok` manually.
- **`gpustat-not-installed` on a GPU node** — `pip install --user gpustat` on
  that node.
- **Master section says "stale!"** — cron stopped, or git push failing. Check
  `/tmp/lsm.log` on the master.
- **PAT expired** — push fails silently. Reissue the PAT, update the remote
  URL: `git remote set-url origin https://<account>:<NEW_PAT>@github.com/...`.

## PAT rotation

Set a calendar reminder to rotate the PAT before the 1-year mark. When you
rotate, update the remote URL on every master.
